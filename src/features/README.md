# Features Directory

This directory contains feature-based modules that group related functionality together.

## Purpose
- Feature-specific components, hooks, and utilities
- Business logic organized by domain
- Self-contained feature modules
- Feature-specific API calls and state management

## Structure
- Each subdirectory represents a feature
- Include components, hooks, types, and utilities for each feature
- Keep features self-contained and modular
- Use index files for clean imports
